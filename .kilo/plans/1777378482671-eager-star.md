# AVH-Align Deepfake Detection Evaluation Plan

## Overview
Evaluate the AVH-Align model on the dataSubset dataset and compute comprehensive metrics (Accuracy, Precision, Recall, F1, ROC_AUC, PR_AUC, EER, FPR).

## Steps Summary

### 1. Environment Setup
- Verify working directory: `C:\t309`
- Check required Python packages: torch, numpy, scikit-learn, pandas
- Create output directory: `C:\t309\results\avh_aligned\`

### 2. Inspect DataSubset
- Read 5 JSON metadata files: av1.metadata.json, dfdc.metadata.json, faceavceleb.metadata.json, faceforensics.metadata.json, lavdf.metadata.json
- Extract video paths and labels (0=real, 1=fake presumed)
- Determine if videos exist as files or if features are already extracted

### 3. Check for Pre-extracted AV-Hubert Features
- Expected feature format: `.npz` files with 'visual' and 'audio' arrays
- Feature naming convention likely: `{video_path_hash_or_id}.npz` or in subdirectories
- The eval.py script expects: `--features_path` pointing to root folder of features
  and `--metadata` pointing to CSV file with ground truth labels

### 4. Extract Missing Features (if needed)
- If features don't exist, need to:
  a. Preprocess videos with `deepfake_preprocess.py`
  b. Extract features with `deepfake_feature_extraction.py` (using AV-Hubert)
- This could be time-consuming; may skip if features already exist

### 5. Prepare Evaluation Input
- Create CSV metadata file compatible with eval.py format:
  - Columns: path (relative to features_path), label
- Convert JSON labels to CSV format

### 6. Run AVH-Align Inference
- Execute: `python eval.py --checkpoint_path checkpoints/AVH-Align_AV1M.pt --features_path <path> --metadata <csv> --dataset combined`
- Capture output scores (alignment scores)

### 7. Compute Metrics
- Use raw alignment scores to compute:
  - **Accuracy**: threshold-based (0.5 after normalization)
  - **Precision**: positive predictive value
  - **Recall**: true positive rate
  - **F1**: harmonic mean of precision/recall
  - **ROC_AUC**: area under ROC curve
  - **PR_AUC**: average precision score
  - **EER**: Equal Error Rate (FPR = FNR)
  - **FPR**: False Positive Rate (at 95% TPR)
- Generate per-dataset breakdown

### 8. Save Results
- Format all metrics into `result.md`
- Include per-dataset tables
- Save to `C:\t309\results\avh_aligned\result.md`

## Key Considerations

### Score Interpretation
From eval.py (line 22): `score = torch.logsumexp(-output, dim=0).detach().cpu().squeeze()`
- Lower scores = more likely REAL (since -output is negated, lower alignment score = better alignment = real)
- For fake detection: need to invert or set threshold appropriately

### Feature Requirements
eval.py expects:
- Features saved as `.npz` files (loaded via `np.load`)
- Each file contains 'visual' and 'audio' tensors
- CSV metadata: path (to .npz relative to features_path) and label

### Metric Computation Strategy
1. Collect all raw scores from AVH-Align
2. Invert scores (multiply by -1) so higher = more likely FAKE
3. Normalize to [0,1] for threshold-based metrics
4. Use sklearn for ROC/PR AUC (works with raw scores)
5. Compute EER from ROC curve (point where FPR = FNR)
6. Compute FPR at 95% TPR
